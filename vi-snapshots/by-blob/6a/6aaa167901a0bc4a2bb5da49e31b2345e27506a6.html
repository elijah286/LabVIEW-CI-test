Timed Loop Actor.lvlib:Timed Loop Actor.lvclass:Update Core.vi

This VI specifies the action that a timed loop controller should take every time the loop executes. Children may put any action they want into their override VIs, but they should be aware that the action should be able to execute within the time specified in "poll rate". Child classes should take care to only include functionality that is deterministic -- no asynchronous functions, no operations that require the UI thread, etc. The Timed Loop Controller class is designed to provide deterministic repetitive behavior when deployed on a real-time system, and any code that isn't deterministic can cause odd system faults that are hard to debug.
