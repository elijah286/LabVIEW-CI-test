Air Cooler Application.lvlib:Air Cooler Application.lvclass:Localization.vi

This VI is used to provide localization -- that is, support for other human languages -- in the user interface. Given an input string, this VI returns the string that should be displayed for the current language. The idea is that you as a developer can use the Export/Import Strings feature of LabVIEW to produce a file that a translator can localize. The convention is that any control with "DNL_" as its label is skipped during localization (in English, this stands for Do Not Localize). The first array in this VI is one of these DNL_ controls. This array has the fixed strings that the program uses. The second array has the translated strings. Using this for all user visible text is sometimes cumbersome on the programmer, but it limits how many VIs have to be reviewed for localization.

Even if you are not translating into other languages, this style of string lookup provides a central location for all your user visible text, which you may reuse at multiple points in your program, thus making it easy for tech writers to review for spelling or to make late-in-development-cycle changes with minimal risk to the rest of the code.
